1. Executive Summary

The AI-Powered Graduate Career Acceleration Platform is an end-to-end intelligent system designed to bridge the gap between academic achievement and professional employment for graduates, with a particular focus on the Cameroonian and broader African market. The platform combines three tightly integrated AI subsystems — an Intelligent Profile Engine, a Semantic Job Matching Engine, and an AI Career Coach — into a unified career enablement ecosystem.

Platform Mission
Analyze → Match → Train → Deploy graduates into relevant employment
Reduce graduate unemployment through intelligent skill visibility and preparation
Provide personalized, continuous career coaching powered by large language models
Serve as a scalable SaaS platform for universities, training centres, and individual graduates

Unlike traditional job boards that provide only listings, this platform actively develops the candidate from profile creation through interview readiness, measuring progress with a proprietary Interview Readiness Index (IRI). The system is built on a modern, fully open-source or free-tier technical stack centred on FastAPI, Next.js, CrewAI, LangChain, and Google Gemini 2.5 Flash.
 
2. Problem Statement & Context

2.1 The Graduate Employment Challenge
Graduates globally — and specifically in Cameroon — face a persistent disconnect between academic achievement and employment readiness. This manifests in four key systemic failures:

Problem	Root Cause	Impact
Skill Misrepresentation	Traditional CVs rely on self-reporting without validation	Recruiters cannot assess true capabilities
Inefficient Job Matching	Keyword-based filtering ignores semantic context	Talented candidates remain undiscovered
Limited Interview Readiness	No structured, personalized preparation pathways	High rejection rates despite relevant skills
Low Recruiter Visibility	Poor profiling systems bury qualified candidates	Prolonged unemployment and underemployment

2.2 Why AI Is the Solution
Artificial intelligence — specifically large language models (LLMs), vector embeddings, and agentic frameworks — enables a platform that can:
•	Extract nuanced skills from unstructured data (GitHub repos, portfolio documents, project descriptions)
•	Perform semantic similarity matching far beyond keyword filtering
•	Simulate realistic interview environments with adaptive questioning
•	Provide continuous, personalized coaching at a scale no human coach could achieve
•	Generate an objective, data-driven Interview Readiness Index (IRI) score
 
3. System Architecture Overview

3.1 High-Level Architecture
The platform follows a microservices-inspired layered architecture with five primary layers communicating over REST APIs and message queues:

Layer	Technology	Responsibility
Presentation Layer	Next.js 14 (React)	User interface, dashboards, interview chatbot UI
API Gateway Layer	FastAPI (Python 3.11+)	REST API, authentication, request routing
Agent Orchestration Layer	CrewAI + LangChain	Multi-agent task orchestration for profile, matching, coaching
AI Inference Layer	Gemini 2.5 Flash (Google AI)	LLM reasoning, generation, and evaluation
Data Layer	PostgreSQL + ChromaDB + Redis	Relational data, vector store, caching

3.2 Subsystem Overview
The platform is decomposed into three primary subsystems, each implemented as a dedicated CrewAI crew:

Subsystem 1 — Intelligent Profile Engine
Ingests: CV/resume files, GitHub profiles, portfolio documents, LinkedIn data, project descriptions
Processes: NLP extraction of skills, tools, domain expertise, project complexity
Outputs: Structured candidate profile, skill vector, skill gap analysis, auto-generated CV
Agents: Document Parser Agent, Skill Extractor Agent, Profile Builder Agent, CV Generator Agent

Subsystem 2 — AI Job Matching Engine
Ingests: Candidate skill vectors, job descriptions from database/scrapers
Processes: Embedding generation, cosine similarity scoring, company fit analysis
Outputs: Ranked job recommendations with explainable match reasoning
Agents: Job Ingestion Agent, Embedding Agent, Matching Agent, Explainability Agent

Subsystem 3 — AI Career Coach
Ingests: Candidate profile, target job role, historical session data
Processes: Mock interview simulation, response evaluation, coaching plan generation
Outputs: Performance feedback, IRI score, personalised learning plans, progress tracking
Agents: Question Generation Agent, Evaluation Agent, Feedback Agent, Progress Tracking Agent

6. CrewAI Agent Architecture

The platform uses three CrewAI Crews, each with dedicated agents and tasks. All agents use Gemini 2.5 Flash as their underlying LLM, accessed via the LangChain-Google-GenAI adapter.

6.1 Crew 1 — Intelligent Profile Engine
Agent Definitions

Agent Name	Role	Goal	Tools Used
DocumentParserAgent	Document Analyst	Extract raw text and structure from uploaded files (PDF, DOCX, TXT)	PDFTool, DocxTool, FileTool
SkillExtractorAgent	Skill Intelligence Analyst	Identify all technical skills, tools, domain knowledge, and project complexity from extracted text	GeminiLLMTool, RegexTool
GitHubAnalysisAgent	Repository Analyst	Analyse GitHub repositories for languages, frameworks, project sophistication, and commit patterns	GitHubAPITool
ProfileBuilderAgent	Career Profile Architect	Synthesise all extracted data into a structured, scored candidate profile with skill gap analysis	GeminiLLMTool, ChromaDBTool
CVGeneratorAgent	Professional CV Writer	Generate an ATS-optimised CV from the structured profile data	GeminiLLMTool, DocxGeneratorTool

Task Flow
Task 1: parse_documents()
  → Input:  List of uploaded file paths
  → Output: Dict {filename: extracted_text}
 
Task 2: extract_skills()
  → Input:  Extracted text + GitHub data
  → Output: List[SkillModel] with confidence scores
 
Task 3: analyse_github()
  → Input:  GitHub username / repo URLs
  → Output: Dict {languages, frameworks, complexity_score}
 
Task 4: build_profile()
  → Input:  Skills list + GitHub analysis + education data
  → Output: StructuredProfile + skill_vector (stored in ChromaDB)
 
Task 5: generate_cv()
  → Input:  StructuredProfile
  → Output: .docx CV file + plain text version

Sample CrewAI Code Structure
from crewai import Agent, Task, Crew, Process
from langchain_google_genai import ChatGoogleGenerativeAI
 
llm = ChatGoogleGenerativeAI(
    model='gemini-2.5-flash',
    google_api_key=settings.GEMINI_API_KEY
)
 
skill_extractor = Agent(
    role='Skill Intelligence Analyst',
    goal='Extract all technical and soft skills from candidate documents',
    backstory='Expert in talent assessment with 10 years in tech recruitment',
    llm=llm,
    tools=[document_tool, github_tool],
    verbose=True
)
 
extract_skills_task = Task(
    description='Analyse the provided text and extract skills as a JSON array',
    expected_output='JSON array of {name, category, proficiency, confidence}',
    agent=skill_extractor
)
 
profile_crew = Crew(
    agents=[parser, skill_extractor, github_analyst, builder, cv_generator],
    tasks=[parse_task, extract_task, github_task, build_task, cv_task],
    process=Process.sequential,
    verbose=True
)

6.2 Crew 2 — Job Matching Engine
Agent Definitions

Agent Name	Role	Goal	Tools Used
JobIngestionAgent	Job Market Analyst	Scrape and normalise job listings from multiple sources into a standard schema	WebScraperTool, JobAPITool
EmbeddingAgent	Vector Intelligence Specialist	Generate and store semantic embeddings for both job descriptions and candidate profiles	SentenceTransformerTool, ChromaDBTool
MatchingAgent	Career Match Strategist	Compute cosine similarity between candidate and job vectors; rank top matches	ChromaDBQueryTool, ScoringTool
ExplainabilityAgent	Recommendation Analyst	Generate human-readable explanations for why each job is a strong or weak match	GeminiLLMTool

Matching Algorithm
# Embedding generation (Sentence Transformers - local, free)
from sentence_transformers import SentenceTransformer
model = SentenceTransformer('all-MiniLM-L6-v2')
 
# Step 1: Embed candidate skill profile
profile_text = ' '.join([skill.name for skill in profile.skills])
profile_vector = model.encode(profile_text)
 
# Step 2: Store in ChromaDB
collection.add(
    embeddings=[profile_vector.tolist()],
    ids=[str(profile.id)],
    metadatas=[{'type': 'candidate'}]
)
 
# Step 3: Query for nearest job embeddings
results = job_collection.query(
    query_embeddings=[profile_vector.tolist()],
    n_results=20
)
 
# Step 4: Re-rank with Gemini for contextual fit
# ExplainabilityAgent generates match_reason for top 10

6.3 Crew 3 — AI Career Coach
Agent Definitions

Agent Name	Role	Goal	Tools Used
QuestionGeneratorAgent	Senior Interviewer	Generate role-specific, adaptive interview questions (behavioral, technical, situational)	GeminiLLMTool, ProfileContextTool
EvaluationAgent	Performance Analyst	Score candidate responses on clarity, accuracy, confidence, and structure using rubrics	GeminiLLMTool, RubricTool
FeedbackAgent	Career Development Coach	Produce actionable, specific improvement feedback for each response	GeminiLLMTool, SessionHistoryTool
IRICalculatorAgent	Readiness Scoring Specialist	Aggregate session scores into an updated Interview Readiness Index score	ScoringAlgorithmTool, DBTool
LearningPlanAgent	Learning Pathway Designer	Create personalised learning plans targeting identified weak areas	GeminiLLMTool, ResourceDBTool

Interview Session Flow
1. Candidate selects a target job role
2. QuestionGeneratorAgent pulls profile + job description
   → Generates 10-15 adaptive questions (mix: 40% technical, 40% behavioral, 20% situational)
3. Question delivered via WebSocket to Next.js chat UI
4. Candidate responds (text or voice-to-text)
5. EvaluationAgent scores response:
   {
     clarity:    0-25,   # Was the answer clear and well-structured?
     accuracy:   0-25,   # Was the technical content correct?
     confidence: 0-25,   # Did the candidate demonstrate confidence?
     relevance:  0-25    # Did the answer address the question?
   }
6. FeedbackAgent generates specific feedback
7. Follow-up question generated if score < 60
8. Session ends → IRICalculatorAgent updates IRI score
9. LearningPlanAgent generates updated study recommendations

Interview Readiness Index (IRI) Formula
IRI = (W1 × Avg_Clarity) + (W2 × Avg_Accuracy) + (W3 × Avg_Confidence) +
      (W4 × Sessions_Completed_Factor) + (W5 × Improvement_Trend)
 
Where:
  W1 = 0.25  (Communication clarity weight)
  W2 = 0.30  (Technical accuracy weight)
  W3 = 0.20  (Confidence weight)
  W4 = 0.15  (Session volume factor — rewards practice)
  W5 = 0.10  (Improvement trend — rewards progress)
 
IRI Range: 0 – 100
  0  – 40:  Not Ready (intensive coaching recommended)
 41  – 65:  Developing (targeted practice areas identified)
 66  – 80:  Nearly Ready (minor refinements needed)
 81  – 100: Job Ready (apply with confidence)

8.3 Authentication Flow
1. User submits login form → POST /api/v1/auth/login
2. Server returns { access_token, refresh_token, user }
3. access_token stored in memory (Zustand store)
4. refresh_token stored in httpOnly cookie
5. All API calls include Authorization: Bearer {access_token}
6. On 401 response → auto-refresh via POST /auth/refresh
7. On refresh failure → redirect to /login
 
// Next.js middleware (middleware.ts) protects /dashboard routes:
export function middleware(request: NextRequest) {
  const token = request.cookies.get('refresh_token');
  if (!token && request.nextUrl.pathname.startsWith('/dashboard')) {
    return NextResponse.redirect(new URL('/login', request.url));
  }
}

9.2 Async Task Queue (Celery)
Long-running AI tasks (profile analysis, job matching) are offloaded to Celery workers to prevent API timeout. Redis serves as both the message broker and result backend.

# tasks/analyse_profile.py
from celery import shared_task
from app.agents.profile_crew import run_profile_crew
 
@shared_task(bind=True, max_retries=3)
def analyse_profile_task(self, user_id: str, document_ids: list):
    try:
        result = run_profile_crew(user_id, document_ids)
        return result
    except Exception as exc:
        raise self.retry(exc=exc, countdown=60)
 
# API endpoint triggers task:
# POST /profile/analyse
task = analyse_profile_task.delay(user.id, document_ids)
return {'task_id': task.id, 'status': 'processing'}
 
# Client polls: GET /tasks/{task_id}/status
# Returns: {status: 'pending'|'processing'|'done'|'failed', result: {...}}

10.2 Query Pattern — Job Matching
import chromadb
 
client = chromadb.HttpClient(host='chromadb', port=8000)
jobs_collection = client.get_collection('job_listings')
 
# Find top 20 jobs for a candidate
results = jobs_collection.query(
    query_embeddings=[candidate_vector],
    n_results=20,
    where={'is_active': True},
    include=['metadatas', 'distances', 'documents']
)
 
# distances = cosine distances (lower = more similar)
# similarity_score = 1 - distance (higher = better match)
matches = [
    {
        'job_id': meta['job_id'],
        'similarity_score': 1 - dist,
        'title': meta['title']
    }
    for meta, dist in zip(results['metadatas'][0], results['distances'][0])
]

13. Security Considerations

Risk	Mitigation Strategy	Implementation
Unauthorized API access	JWT Bearer token authentication	python-jose, bcrypt, short-lived access tokens (30 min)
SQL Injection	ORM-based queries only	SQLAlchemy with parameterised queries — no raw SQL
File upload abuse	File type validation + size limits	python-magic for MIME type check, 10MB max per file
Sensitive data exposure	Environment variables for secrets	.env not committed to Git, Docker secrets in production
CORS misconfiguration	Strict CORS origin whitelist	FastAPI CORSMiddleware with explicit allow_origins list
Rate limiting abuse	Request rate limiting per user	slowapi (FastAPI rate limiter) on auth and AI endpoints
API key leakage	Server-side only key usage	Gemini API key never sent to frontend, only used in backend
